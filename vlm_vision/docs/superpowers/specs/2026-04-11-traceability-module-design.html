1. Purpose & Business Goals

Build a traceability system that captures video evidence of the entire packing-to-shipping process at Metwall. The system generates verifiable proof that products were correctly packed, labeled, palletized, and loaded onto trucks.

Business Objectives

  • Visual evidence — Generate timestamped video proof of every shipment
  • Reduce claims — Prove items were correctly packed with irrefutable footage
  • Protect revenue — Defend against non-payment disputes with loading evidence
  • Catch errors early — Dual barcode verification (camera + scanner) detects labeling mistakes before shipping

2. System Architecture

2.1 Three-Process Modular Design

The traceability module runs as three separate processes sharing a core library. This isolates video recording (resource-intensive) from AI inference and keeps the existing VLM Bay Agent untouched.

Process 1 — VLM Bay Agent

Existing — no changes

  • CameraAgent (bay cameras)
  • Detector (metwall.onnx)
  • PickVerifier, ModulaClient
  • Display overlay (WebSocket)

Process 2 — Traceability Agent

New module

  • ZoneManager (5 zones)
  • BarcodeReader (camera + USB)
  • PalletTracker
  • EventCorrelator
  • EvidenceGenerator
  • Detector (3 ONNX models)

Process 3 — Video Recorder

New module

  • MultiCameraRecorder
  • VideoSegmenter (5-min chunks)
  • CloudUploader
  • RetentionManager (60 days)
  • ClipExtractor (±30s evidence)

Shared Core Library

Refactored from existing modules: camera_agent, detector, offline_queue, cloud_sync_client, config, models, display_server

5. Barcode Dual Verification

Zone 2 performs dual verification using two independent barcode reads:

Camera Path

  1. Capture frame from Zone 2 camera
  2. YOLO barcode detection (barcode.onnx)
  3. Crop barcode region from frame
  4. Decode with pyzbar/zxing
  5. Result: barcode_camera value

Scanner Path

  1. Physical USB barcode scanner
  2. Serial/HID input listener
  3. Timestamp correlation
  4. Result: barcode_scanner value

EventCorrelator matches readings within a 5-second window:

ResultConditionAction
MATCHBoth readings agreebarcode_verified = true
MISMATCHDifferent valuesAlert event + automatic evidence clip generation
PARTIALOnly one source readWarning logged, camera_only or scanner_only status

6. Video Pipeline

6.1 Three-Stage Flow

Stage 1: Local Recording

H.264 encoding via OpenCV VideoWriter. Split into 5-minute segments per camera. Local buffer: 24 hours.

Stage 2: Cloud Upload

Background queue uploads completed segments to Catalyst File Store. Retry on network failure. Non-blocking to recording.

Stage 3: Evidence Clip Extraction

On-demand: user searches event → system locates video_segment + offset → extracts ±30s clip → clip uploaded and retained indefinitely.

6.3 Retention Policy

  • Video segments: 60 days in cloud, auto-deleted by RetentionManager
  • Local buffer: 24 hours on factory machine, then cleaned up
  • Evidence clips: Retained indefinitely (flagged disputes never expire)
  • Event metadata: Kept forever in Catalyst Datastore
Resilience (Offline-First) Recording never stops, even if cloud is unreachable. Segments queue locally and upload when connection restores. 24-hour local buffer ensures no data loss. Same offline-queue pattern as existing VLM pick sync.

8. Admin Dashboard

Three new tabs added to the existing VLM admin dashboard:

8.1 Traceability Search

  • Search bar: barcode, pallet ID, order/shipment number, date range
  • Results table: barcode, SKU, pallet, shipment, zones passed (N/5), barcode match status, date
  • Mismatches highlighted in red with auto-generated evidence clips
  • Actions: “Timeline” link and “Clips” link per row

8.2 Timeline View

  • Visual vertical timeline showing a box’s journey through all 5 zones
  • Each zone: timestamp, event details, confidence scores
  • Clickable “Play Clip” button per zone → plays ±30s evidence clip in-browser
  • Color-coded by zone (blue, green, orange, red, purple)

8.3 Live Zones

  • Grid of 5 zone cards with real-time MJPEG camera feeds
  • Each card: zone name, camera feed, current box/pallet info, status
  • Same WebSocket pattern as existing bay overlay but multi-zone

8.4 Video Archive

  • Browse stored video segments by camera, zone, date
  • Upload status indicators (uploaded/pending/failed)
  • Manual clip extraction trigger
  • Storage usage statistics

12. Testing Strategy

  • Unit tests for each new module (zone_manager, barcode_reader, pallet_tracker, event_correlator, evidence_generator, video_segmenter, clip_extractor, retention_manager)
  • Integration tests for barcode dual verification flow
  • Integration tests for event → video segment → clip extraction pipeline
  • Cloud function tests for all new routes
  • End-to-end test with mock cameras simulating full zone traversal
  • Same mock-first (London School TDD) approach as existing tests

13. Out of Scope

  • Customer-facing portal (internal search only for now)
  • Multi-factory deployment (single installation)
  • Real-time streaming to cloud (only recorded segments uploaded)
  • Automatic dispute filing (manual process using evidence clips)
  • Integration with ERP/WMS beyond existing Modula client
