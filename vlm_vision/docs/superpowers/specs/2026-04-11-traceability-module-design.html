1. Purpose & Business Goals

Build a traceability system that captures video evidence of the entire packing-to-shipping process at Metwall. The system generates verifiable proof that products were correctly packed, labeled, palletized, and loaded onto trucks.

Business Objectives

  • Visual evidence — Generate timestamped video proof of every shipment
  • Reduce claims — Prove items were correctly packed with irrefutable footage
  • Protect revenue — Defend against non-payment disputes with loading evidence
  • Catch errors early — Dual barcode verification (camera + scanner) detects labeling mistakes before shipping

2. System Architecture

5. Barcode Dual Verification

Zone 2 performs dual verification using two independent barcode reads:

Camera Path

  1. Capture frame from Zone 2 camera
  2. YOLO barcode detection (barcode.onnx)
  3. Crop barcode region from frame
  4. Decode with pyzbar/zxing
  5. Result: barcode_camera value

Scanner Path

  1. Physical USB barcode scanner
  2. Serial/HID input listener
  3. Timestamp correlation
  4. Result: barcode_scanner value

EventCorrelator matches readings within a 5-second window:

ResultConditionAction
MATCHBoth readings agreebarcode_verified = true
MISMATCHDifferent valuesAlert event + automatic evidence clip generation
PARTIALOnly one source readWarning logged, camera_only or scanner_only status

6. Video Pipeline

6.1 Three-Stage Flow

Stage 1: Local Recording

H.264 encoding via OpenCV VideoWriter. Split into 5-minute segments per camera. Local buffer: 24 hours.

Stage 2: Cloud Upload

Background queue uploads completed segments to Catalyst File Store. Retry on network failure. Non-blocking to recording.

Stage 3: Evidence Clip Extraction

On-demand: user searches event → system locates video_segment + offset → extracts ±30s clip → clip uploaded and retained indefinitely.

6.3 Retention Policy

  • Video segments: 60 days in cloud, auto-deleted by RetentionManager
  • Local buffer: 24 hours on factory machine, then cleaned up
  • Evidence clips: Retained indefinitely (flagged disputes never expire)
  • Event metadata: Kept forever in Catalyst Datastore
Resilience (Offline-First) Recording never stops, even if cloud is unreachable. Segments queue locally and upload when connection restores. 24-hour local buffer ensures no data loss. Same offline-queue pattern as existing VLM pick sync.

14. Overview — Automated VLM Picking

Phase 2 adds a collaborative robot arm to one Modula VLM bay, replacing the human operator for the picking (kitting) stage. The robot uses VLM Vision’s existing camera + AI detection as its eyes — Phase 1 builds the robot’s vision, Phase 2 adds the arms.

Key Insight: Phase 1 Builds the Robot’s Eyes The same camera and YOLO model that verifies human picks in Phase 1 becomes the robot’s guidance system in Phase 2. Detection bounding box → tray slot coordinates → arm trajectory.

15.3 End Effector (Gripper)

  • 500mm vacuum bar with 3–4 suction cups (grips center section of profile)
  • Venturi vacuum generator (no external compressor needed)
  • Vacuum pressure sensor confirms grip before lifting
  • Foam/rubber cups — won’t scratch aluminum surface
  • Quick-release tool changer for future gripper swaps

15.4 Vertical Dolly Rack

  • Wheeled rack with locking casters, 10–15 vertical slots
  • Rubber-padded dividers, slot width 25mm
  • Height: 3.5m (accommodates 3.3m profiles)
  • Slot occupancy sensors (optical) confirm placement
  • Rack ID barcode for traceability
  • Operator rolls loaded rack to packing zone

16. Pick Sequence

16.1 Three-Step Pick Cycle

Step 1 — Camera Detects

~0.5 seconds

  • Modula presents tray
  • VLM Vision camera captures frame
  • YOLO detects SKU + bounding box
  • Bbox → tray slot XY position
  • Pick command sent to arm

Step 2 — Arm Picks + Rotates

~4.5 seconds

  • Arm moves to tray slot XY
  • Z lowers to profile height
  • Vacuum engages (pressure confirms)
  • Lifts clear of tray dividers
  • Wrist rotates 90° (flat → vertical)

Step 3 — Place in Rack

~3 seconds

  • Arm traverses to open rack slot
  • Lowers profile into slot
  • Vacuum releases
  • Slot sensor confirms placement
  • Arm returns to home
Total cycle: ~8 seconds per pick = ~450 picks/hour Compared to human operator averaging ~20–30 seconds per pick (~120–180/hour), the robot delivers 2.5–3.75x throughput improvement.

18. Calibration & Setup

18.1 Camera-to-Robot Calibration

One-time procedure to map camera pixels to robot arm coordinates:

  1. Place calibration target (checkerboard) on VLM tray
  2. Camera captures image → detect checkerboard corners
  3. Robot arm touches each corner → record arm XYZ position
  4. Compute homography matrix (pixel → mm transformation)
  5. Store calibration in calibration.json
  6. Re-calibrate if camera or arm is repositioned

18.2 Tray Mapping

  • Modula VLM trays have fixed divider positions
  • Map each divider configuration to slot coordinates
  • Store in tray_configs.json — one entry per tray layout
  • When Modula presents a tray, its ID identifies the configuration

19. Safety

  • Cobot safety: UR20/FANUC CRX have built-in force/torque limiting — auto-stops on human contact (ISO 10218-1, ISO/TS 15066)
  • No safety cage required — cobots are rated for human collaboration
  • Swing clearance: 1.5m radius around arm base marked with yellow floor tape
  • E-stop buttons: One on robot controller, one wall-mounted
  • Vacuum fail-safe: If pressure drops during transit, arm stops and holds position
  • Light indicator: Green (running), yellow (paused), red (error/e-stop)
  • Speed reduction: 50% max speed when proximity sensor detects human
