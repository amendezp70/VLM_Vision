1. Purpose & Business Goals

VLM Vision is an AI-powered system for Metwall’s Modula VLM warehouse operations. It combines real-time pick verification, continuous video evidence, and automated robotic picking.

  • Verify picks — AI camera confirms the correct part is taken from the correct tray
  • Video evidence — Continuous recording creates timestamped proof for dispute resolution
  • Automate picking — Cobot arm replaces manual picking of 3m aluminum profiles (2.5–3.75x throughput)
  • Offline resilience — System never stops; data syncs when connection restores

Camera Setup

  • Platform-aware: AVFoundation (macOS), DirectShow (Windows), V4L2 (Linux)
  • Resolution tiers: 4K → 1080p → 720p (auto-fallback)
  • Frame rate: 10 FPS default, configurable

Detection (metwall.onnx)

ParameterValue
ModelYOLOv8n, 150 SKU classes
Class formatSKU__color (e.g., STL-P-100-BK__black)
Input640×640px
Confidence threshold0.25
IOU threshold0.45

Pick Verification Flow

  1. Modula WMS sends active pick order
  2. Camera captures “before” frame → run detection
  3. Wait 1.5 seconds for pick action
  4. Capture “after” frame → run detection
  5. Compare SKU counts: correct, short, or wrong
  6. PickEvent queued → synced to cloud

Continuous Recording & Evidence Clip Extraction

6. Video Pipeline

Three-stage flow: record locally → upload to cloud → extract clips on demand.

Stage 1: Local Recording

H.264 via OpenCV VideoWriter. 1920×1080 at 15 FPS (~2 Mbps). Split into 5-minute segments. Filename: cam{id}_{YYYY-MM-DD}_{HH-mm-ss}.mp4

Stage 2: Cloud Upload

Background queue uploads completed segments to Catalyst File Store. Retry with exponential backoff on failure. Non-blocking to recording.

Stage 3: Evidence Clip Extraction

On-demand: user searches event → system locates video_segment + offset → extracts ±30s clip → uploaded and retained indefinitely.

10. Pick Sequence

10.1 Three-Step Cycle

Step 1 — Detect

~0.5 seconds

  • Modula presents tray
  • Camera captures frame
  • YOLO detects SKU + bbox
  • Bbox → tray XY
  • Command sent to arm

Step 2 — Pick + Rotate

~4.5 seconds

  • Arm moves to tray slot
  • Z lowers, vacuum engages
  • Pressure confirms grip
  • Lifts clear of dividers
  • Wrist rotates 90°

Step 3 — Place

~3 seconds

  • Traverse to rack slot
  • Lower into slot
  • Vacuum releases
  • Sensor confirms
  • Arm returns home
~8 seconds per pick = ~450 picks/hour Human operator averages 20–30s/pick (~120–180/hr). Robot delivers 2.5–3.75x throughput improvement.

11.4 Calibration

  1. Place checkerboard on VLM tray
  2. Camera detects corners in pixels
  3. Robot arm touches each corner → record XYZ
  4. Compute homography matrix (pixel → mm)
  5. Store in calibration.json
  6. Re-calibrate if camera or arm repositioned

11.5 Dashboard: Robot Status Panel

  • Real-time arm position (joint angles)

  • Vacuum pressure gauge
  • Rack slot occupancy grid
  • Pick rate graph (current vs. average)
  • Error log with video timestamps
  • Manual controls (pause, resume, home, e-stop)
